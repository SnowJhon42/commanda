# Cambios Locales

Objetivo: registrar mejoras y cambios pendientes de subir, sin desplegar nada hasta que se decida un release conjunto.

## Estado

- Todo lo listado en esta carpeta puede estar solo en local.
- Nada de esta carpeta implica deploy.
- Cuando se quiera subir, esta carpeta sirve como checklist de consolidacion.

## Convencion

- Un archivo por cambio o bloque de cambios.
- Nombre sugerido: `NNN_YYYY-MM-DD_tema.md`
- Numeracion incremental: `001`, `002`, `003`, etc.

## Formato sugerido

```md
# NNN - Titulo corto

Fecha: YYYY-MM-DD
Estado: LOCAL | LISTO_PARA_SUBIR | SUBIDO

## Resumen

Breve descripcion del cambio.

## Alcance

- backend
- client
- staff
- docs

## Archivos tocados

- ruta/archivo

## Validacion local

- que se probo

## Pendiente antes de subir

- checks o mejoras faltantes
```
