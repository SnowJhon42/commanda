# COMANDA - Website Premium Spec

Fecha: 2026-04-16
Owner: CTO-Agent / Marcos
Estado: base estrategica aprobable

## Objetivo
Definir una web publica premium para COMANDA orientada a bares y restaurantes, con foco comercial en:
- orden operativo
- transparencia del servicio
- menos errores
- mejora de tiempos
- mejor experiencia del cliente
- feedback util para el establecimiento
- difusion organica del lugar por parte de los clientes

CTA principal del sitio:
`Agendar demo`

## Audiencia
La web debe hablar primero a:
- dueños de restaurantes y bares
- socios
- gerentes
- encargados de salon
- perfiles operativos que necesitan mas control sin volver mas compleja la atencion

## Posicionamiento
COMANDA no debe presentarse como un "menu QR" ni como una web de software generica.

La idea central es:
`COMANDA ordena la operacion, hace mas transparente el servicio y mejora la experiencia del cliente desde la mesa. Ademas, convierte una buena experiencia en feedback real y en difusion organica para el local.`

## Propuesta de valor
`COMANDA ayuda a restaurantes y bares a organizar el flujo de pedidos, reducir errores, mejorar tiempos de atencion y dar visibilidad durante el servicio. Ademas, permite captar feedback real del cliente y facilitar que una buena experiencia se comparta de forma organica.`

## Jerarquia de mensaje
El sitio debe argumentar en este orden:
1. Problema operativo real
2. Orden y transparencia como solucion principal
3. Menos errores y mejor manejo de tiempos
4. Mejor experiencia para el cliente
5. Feedback como activo del negocio
6. Difusion organica como diferenciador comercial

La regla es simple:
- primero vender eficiencia y control
- despues vender experiencia
- al final mostrar feedback y crecimiento organico como ventaja distintiva

## Tono verbal
Tono base:
- premium cercano
- claro
- seguro
- comercial
- sobrio

Evitar:
- promesas vacias de innovacion
- exceso de jerga tecnica
- claims sin prueba
- tono corporativo frio
- copy tipo SaaS generica

## Claims que no se deben inventar
- logos de clientes
- testimonios
- porcentajes de mejora
- "lider del mercado"
- "revoluciona la gastronomia"

## Concepto rector
`Mas orden en la operacion. Menos errores en el servicio. Mejor experiencia en cada mesa.`

## Sistema visual
La web debe sentirse hospitality premium, no dashboard SaaS.

### Direccion visual
- base clara y calida, no blanco clinico
- contraste alto con bloques oscuros para secciones clave
- composicion con aire y ritmo editorial
- mezcla de sobriedad operativa con detalle aspiracional

### Paleta sugerida
- `Fondo claro`: marfil tostado `#F5F0E8`
- `Fondo secundario`: arena `#E8DDCF`
- `Texto principal`: espresso `#221C17`
- `Texto secundario`: nogal `#5C4D42`
- `Acento premium`: cobre oscuro `#A45A3A`
- `Acento operativo`: oliva profundo `#3F4B3C`
- `Bloque oscuro`: carbon cafe `#181411`

### Tipografia sugerida
Titulares:
- `Cormorant Garamond`
- `Bodoni Moda`
- `Fraunces`

Texto y UI:
- `Manrope`
- `Sora`
- `Plus Jakarta Sans`

Regla:
- serif con caracter para promesa y secciones
- sans limpia para legibilidad y botones

### Fotografia y mockups
Usar:
- mesas reales
- luz calida
- manos, platos, celulares, momentos de servicio
- mockups de producto integrados al contexto

Evitar:
- renders genéricos de SaaS
- dashboards flotando sobre gradientes random
- iconografia excesiva

### Movimiento
Animaciones discretas:
- fade y rise en hero
- aparicion escalonada de beneficios
- parallax minimo o desplazamientos suaves
- transiciones lentas y seguras

Evitar microanimaciones de juguete.

## Layout
### Hero
- grilla de 2 columnas en desktop
- copy a la izquierda
- mockup compuesto o escena visual a la derecha
- KPI chips solo si luego existen datos reales

### Secciones intermedias
- alternar bloques claros y oscuros
- usar cambios de ritmo: texto + lista, texto + mockup, cards, timeline
- no encadenar secciones iguales

### Bloque de feedback y difusion
- tratarlo como diferencial premium, no como pie de pagina
- usar una escena o composicion especial
- destacar que el valor sigue despues del pedido

### CTA final
- visualmente mas sobrio que el hero
- foco absoluto en conversion
- sin distracciones ni exceso de links

## Componentes clave
- navbar simple con CTA persistente
- botones primario y secundario
- cards de beneficios
- timeline de funcionamiento
- bloque comparativo problema/solucion
- acordeon FAQ
- mockup frame del producto
- seccion testimonial vacia hasta tener prueba real
